Alur Kerja Staf Lapangan

  1. Security Gate-In: Daftarkan plat nomor truck dan nomor surat jalan supir saat masuk gerbang melalui form registrsi.
  2. Warehouse Staff: Ubah status menjadi Service kala barang mulai dibongkar/dimuat di Row.
  3. Warehouse Selesai: Staf gudang men-set status ke Done bila dokumen/muatan selesai disisihkan.
  4. Security Gate-Out: Security menekan tombol Clear Out untuk mencatat waktu keluar, menghitung selisih durasi otomatis, dan menyimpannya.

Konfigurasi Database Supabase

Hubungkan sistem ke sandboxing awan Anda sendiri

Sistem ini mendukung integrasi database Supabase secara instan. Di bawah ini Anda dapat mengonfigurasi koneksi langsung. Jika tidak dikonfigurasi, sistem akan otomatis menggunakan Mode Demo (menulis ke LocalStorage, merespons aksi secara real-time).

Koneksi berhasil disimpan! Sistem beralih ke basis data cloud.
Status Konektibilitas:

PostgreSQL / SQL Schema

Skema tabel \`vehicles\` untuk database Anda

SQL Editor 
CREATE TABLE vehicles (
  id BIGSERIAL PRIMARY KEY,
  nomor_kendaraan VARCHAR(100) NOT NULL,
  nama_supir VARCHAR(255) NOT NULL,
  nomor_surat_jalan VARCHAR(255) NOT NULL,
  id_row VARCHAR(100) NOT NULL,
  tanggal DATE DEFAULT CURRENT_DATE,
  time_in TIMESTAMPTZ DEFAULT NOW(),
  time_out TIMESTAMPTZ,
  status_bongkar_muat VARCHAR(50) DEFAULT 'Pending',
  durasi VARCHAR(255),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Buka izin RLS penuh untuk sandboxing mudah:
ALTER TABLE vehicles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "RLS Full Access" ON vehicles FOR ALL USING (true) WITH CHECK (true);

-- AKTIFKAN REPLIKASI REAL-TIME WAJIB (PENTING):
ALTER PUBLICATION supabase_realtime ADD TABLE vehicles;

Paling Penting (Realtime replication):

Supabase tidak mereplikasi pembaruan secara real-time secara default untuk alasan performa. Pastikan Anda menjalankan perintah ALTER PUBLICATION supabase_realtime ADD TABLE vehicles; untuk mengaktifkan sinkronisasi real-time instan!
